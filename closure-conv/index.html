Proving Properties of Typed Closure Conversion in Abella

This page contains the formalization of typed closure conversion (TCC) for simply-typed lambda calculus (STLC) and its properties in Abella. TCC is encoded by using the higher-order abstract syntax (HOAS). The source language and target language are both encoded in the HOAS style. Two major properties of TCC are formalized in Abella:

  • TCC preserves types. The term output by TCC has the same type as the input term. The formalization is here.

  • TCC preserves meaning. The term output by TCC simulates the input term under evaluation. The simulation is defined as logical relations between source and target terms. The proof is based on the "Typed Closure Conversion" technical report by Y. Minamide, et al. The formalization is here.

    In the previous proof, the simulation relation assumes termination of evaluations for the source and target language. However, the termination property might not hold for languages richer than STLC. For this, the meaning preservation is proved by using an alternative definition of simulation that does not assume termination. The formalization is here.
